PROMPT 5: OTP AUTHENTICATION SYSTEM
================================================================================

Create complete OTP authentication for OUAGA CHAP.

Components needed:
1. OtpCode model/migration (phone, code (hashed), purpose, attempts, expires_at)
2. OtpService class:
   - generateOtp(phone) - creates 6-digit OTP, stores hashed, returns plain
   - verifyOtp(phone, code) - validates and marks as used
   - Max 3 attempts, 5-minute expiry
3. AuthController:
   - POST /auth/otp/request - send OTP via SMS
   - POST /auth/otp/verify - verify and return token + user
   - POST /auth/logout - revoke token
   - GET /auth/me - current user
4. Form Requests with validation
5. Rate limiting: 3 OTP requests per 15 minutes

SMS sending should use a job (SendOtpSms) for async processing.
For MVP, just log the OTP instead of actually sending SMS.

- pickup_address, pickup_latitude, pickup_longitude (required)

- dropoff_* (same as pickup)
- package_description (optional), package_size (enum)

PROMPT 7: COURIER ORDER MANAGEMENT
================================================================================

Create courier order management for OUAGA CHAP.

CourierController methods:
1. getAvailableOrders(latitude, longitude, radius) - find pending orders nearby
2. acceptOrder(uuid) - assign order to courier
3. updateStatus(uuid, status) - update order status (picked_up, delivered)
4. cancelOrder(uuid, reason) - cancel with penalty tracking
5. getActiveOrder() - get courier's current order
6. updateLocation(latitude, longitude) - update courier position

- Status transitions must be valid (accepted -> picked_up -> delivered)
- 3 cancellations = temporary suspension

Events to dispatch:
- OrderAccepted - notify customer
- OrderPickedUp - notify customer
- OrderDelivered - notify customer, update stats, credit wallet
- CourierLocationUpdated - for tracking
